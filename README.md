# Weather App — PHP + OpenWeatherMap

A simple PHP CLI app that fetches real-time weather data using the OpenWeatherMap API and Guzzle HTTP client.

---

## Requirements

- PHP 8.0+
- Composer
- OpenWeatherMap API Key → [Get one here](https://openweathermap.org/api)

---

## Project Structure

```
Latihan Pweb/
├── src/
│   └── Weather.php       # Weather class
├── vendor/               # Auto-generated by Composer (don't commit)
├── Call.php              # Entry point
├── composer.json
├── .env                  # Your API key (don't commit)
├── example.env           # Template for others
└── .gitignore
```

---

## Installation

**1. Clone the repository**
```bash
git clone https://github.com/your-username/your-repo-name.git
cd your-repo-name
```

**2. Install dependencies**
```bash
composer install
```

**3. Set up your environment file**
```bash
cp example.env .env
```

Then open `.env` and replace with your real API key:
```
WEATHER_API_KEY=your_api_key_here
```

---

## Usage

Run the app from the project root with a city name as argument:
```bash
php Call.php <nama_kota>
```

**Example:**
```bash
php Call.php Surabaya
```

**Example output:**
```
Mendapatkan data cuaca untuk Surabaya ...
Data cuaca untuk Surabaya:
Kota       : Surabaya
Suhu       : 31.5°C
Deskripsi  : few clouds
Kelembapan : 74%
```

If you run it without a city name:
```bash
php Call.php
```
```
Penggunaan Yang Benar : php Call.php <nama_kota>
```

---

## How It Works

1. `Call.php` reads the city name from the command line argument (`$argv[1]`)
2. Creates a `Weather` object and calls `getweather($city)`
3. `Weather.php` uses Guzzle to send a GET request to the OpenWeatherMap API
4. The response is decoded and printed to the terminal

---

## Security

- Never commit your `.env` file or API key
- `.gitignore` excludes `vendor/` and `.env` automatically
- Use `example.env` as a template — it contains no real credentials
